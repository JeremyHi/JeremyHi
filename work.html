propheseer - unified prediction market api

a unified rest api that aggregates prediction market data from polymarket, kalshi, and gemini into a single, normalized format. one api key provides access to 2,400+ markets across all major prediction platforms.

built with an express.js backend and next.js frontend, the platform offers real-time market data via rest api and websockets, cross-platform arbitrage detection, and tiered api plans for different usage levels. designed to reduce integration time from hours to minutes for developers building trading bots, research tools, or media applications.

propheseer marketing automation - self-improving twitter engagement

an autonomous system that grows propheseer's twitter presence by engaging with the prediction market community. the system monitors twitter for relevant conversations about polymarket, kalshi, and trading strategies, then responds with targeted marketing messages designed to drive site engagement and api signups.

the architecture follows a learn-and-adapt loop: scheduled posting windows throughout the day generate structured logs capturing every interaction - target account, message template, engagement metrics. a daily analytics job fetches results and creates performance reports. weekly, an ai model analyzes patterns across accounts, timing, and content styles to identify what drives the most engagement.

what makes it interesting is the closed feedback loop. the system proposes strategy changes, validates them with dry-runs to prevent config breaks, then auto-applies updates to template weights and target priorities. it literally teaches itself better marketing over time, with full transparency through documented learnings.

jhipro.com - vital climbing gym occupancy tracker

a real-time occupancy tracking dashboard for vital climbing gym in brooklyn. built with a python backend and data visualization using plotly, the site displays current gym capacity and historical trends to help climbers plan their visits.

the application provides live occupancy data with historical patterns over multiple time periods (7/30/90 days, yearly) in a clean dark mode interface. it helps the climbing community avoid peak hours and find the best times to visit based on actual usage data.
